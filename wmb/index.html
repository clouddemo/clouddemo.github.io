Introduction


WMB (Workcell Microservices Builder) is an innovative project that provides a modern loose-coupled and high-robustness solution that optimized for IoT domain and devices. It consists of a OT low-code platform (WMB), a microservices builder, an information model (CREEM) and a light-weighted runtime (SPIDR).

The intuitive engineering paradigm of the WMB OT Low-code is FBTD (Function Block Typed Diagram) sourced from an improved and simplified version of IEC61499 based on CREEM that is compatible with Behavior Tree and OPC UA information model.

  • Basic features are ready and can run Danfoss workflow as a sample
    • Low-code programming as a IOT engineering tool.
    • A CIM (CREEM) to support engineering and runtime.
    • Ecosystem via the CREEM
    • SPIDR can parse the CREEM and run in a IPC or IoT device.
  • The workflow includes human, machine, PLC, robot, readers and executors
  • A simple 3D visualization to preview the cyclic
  • WMB can be deployed on shop floor to a real Danfoss sample workstation

Contents of the website include "System and software architectures", "Low-code paragidm designs", "Showcase and demos" and "FB libraries". It's targeting internal stakeholders, developers, testers, reviewers and other relevant colleagues.

WMB in the Overall Strategy
WMB is owned by Siemens Advanta Group. It's part of IT/OT (Information Technology and Operation Technology) Polymerization strategy using Data Layer (DL). It will provide OT monitoring data source for DL. At the same time, enable OT low-code programming for both OT and IT engineers.
