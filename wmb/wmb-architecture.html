WMB System Architecture

Workcell Microservice Builder (WSB) is an innovative architectural solution. The solution provides a modern loose-coupled and high-robustness platform that optimized for IoT network and devices.

The core components of the solution include:

  • Workflow Microservices Builder (WMB)
  • Siemens Python Interpreter-Driven Runtime (SPIDR)
  • Common Runtime-Engineering Exchange Model (CREEM)
  • OT Workcell Microservices (Generated by WMB)
WMB and Data Layer

The Data Layer will be the core of IT/OT Amalgamation. The dashed lines stand for the data flow while the solid lines for invoking flow. All components of WMB will not deal with monitoring data themselves. Instead, the data will be collected and initially filtered by HAL in RA and CREEM OEM Native drivers and send to Data Layer for persistency and data analysis/governance. At the same time, service registration and discovery and related microservice management patterns like circuit breaker will also managed by Data Layer. The concurrency use cases will be supported by related frameworks inside of it. Therefore, Data Layer will be the core of IT/OT Amalgamation. In single word, WMB will be a key part to get through the OT devices / workflows and IoT monitoring data collection layer for the overall Data Layer strategy.

OT Workcell Microservices

Once the CREEM workflows are deployed into Interpreter-Driven Runtime, they will be in idle state. OT Workcell Microservice created by ASB will be the interfaces to IT systems to trigger the sleeping workflows. All of the micrtoservices are container-based and managed by Service Governance sub-system of Data Layer. Since there is no data persistance inside the microservices, WMB and the microservice shall work with Data Layer to enable the capability to support data virtualization for different data sources. There are two typical types of the generated Microservices. The difference is whether the Interpreter-Driven Runtime is inside the Microservice or outside. Type one is more light-weighted while type two can work independently.

History & Source of WMB Project
The domain idea of WMB project is sourced from CCT ARO C2 projects including AppComposer (C.2) and MATE (C.1). But all the architectures are redesigned and all features are developed from scratch. I worked for both of the two projects as Architect, but for now WMB is a service-oriented solution for Siemens Advanta and is separated from the CCT ARO Module C project.
