Openness & Ecosystem of WMB
  • Frontend OT Low-code
    • Real Modularity: Every component can build & run independently
    • Cross Platform: PWA & Electron - Windows / Mac / Linux / iOS / Android...
    • Cloud Native / Serverless Deployment on Public Cloud (AWS)
    • React Component to keep the possibility to be integreated to other platform (e.g. Mendix)
    • Design Language and retina screen support
    • All frontend elements are text-based artifacts (CREEM)
    • Multiple languages support
    • Responsive designed for different resolution / devices; Support touch screen
    • A BFF (backend for frontend) layer
    • A Microservice builder to build OT Workcell Microservices
    • Modern DevOps and CI/CD

  • CREEM Openness
    • All elements exchanging between different sub-systems are CREEM
    • A CREEM editor is provided for Siemens Consultant to create new FBs easily via web panel.
    • Ecosystem using CREEM FBs via indexable metadata
    • The CREEM instances aggregations can be part of Data Layer for a domain data collection
    • Schemas to verify all CREEM instances automatically
    • Specification based on Schemas can be standardized
    • Compatible with Behavior Tree and OPC UA information model
    • OT Workcell Microservices are generated automatically from CREEM
    • Possiblity to be open-sourced

  • SPIDR (v1 or v2) for WMB
    • Light-weighted and even support IoT device deployment
    • Using Python interpreter plus a message broker to provide event-based execution engine
    • Ecosystem based on Python community
    • Contribute to open-source community as Siemens
    • A HMI Control Panel in a Blackbox to control the SPIDR workflow and other features
    • A node-link assembly as the intermediate language (Compiler concept)
    • Possibility to support a Behavior Tree parser for better performance
    • Using OT Workcell Microservices to be invoked by other IT low-code tools
    • SPIDR itself is a Microservice
